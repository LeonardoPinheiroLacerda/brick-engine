== Architectural Purpose

The `GameView` class acts as the primary View controller bridging logical engine commands to DOM behaviors. By isolating native HTML mutations, Layout calculations, and Button listener bindings away from the standard Canvas execution loops, it guarantees the graphical application skeleton scales and interacts seamlessly matching CSS constraints, without corrupting cross-session physics data.

== Features

*   **Scaffolding**: Dynamically injects layouts and elements via components like `ContainerLayout` and `Canvas`.
*   **Aesthetics**: Responsible for dispatching standard dimension calculations (`dimensions.ts`) and theme injections (`applyColors.ts`).
*   **Orchestration**: Manages auxiliary visual tools such as `xref:../modules/Debugger.adoc[Debugger]` and `xref:../modules/SessionModal.adoc[SessionModal]`.

=== `build(): { canvas: p5.Element; canvasWidth: number; canvasHeight: number }`

Builds the visual structure by appending structural layers, generating physical buttons, and initializing canvas metrics.

**Returns**

`{ canvas: p5.Element; canvasWidth: number; canvasHeight: number }` - A strictly bound dimensions object mapping physical canvas bounds.

**Behavior**

This is the dominant method for defining the front-end skeleton. It creates all buttons to subsequently bind them in later pipeline phases. Caches the canvas return parameter avoiding redundant builds (querying `isBodyBuilt()` internal state).

== Controls Binding

=== `bindControls(control: Control): void`

Pairs DOM click events with logical Engine actions.

**Parameters**

[cols="1,1,3", options="header"]
|===
|Name |Type |Description
|`control` |`xref:../interfaces/modules/Control.adoc[Control]` |The active control module resolving interactions.
|===

**Behavior**

Instantiates a `ControlInputHandlerHelper` mapping each isolated layout button (from `_upBtn`, `_actionBtn`, to specific targets like `_resetBtn`) to their respective `ControlKey` actions using native `mousePressed` and `mouseReleased` hooks. Supports pointer displacement logic mapping `mouseOut` as automatic release handlers to prevent stuck events.


'''

=== `unbindControls(): void`

Completely nullifies event listeners on all tracked buttons, preventing interactions when the execution stack prohibits it (e.g., stopping the engine entirely, removing focus).
