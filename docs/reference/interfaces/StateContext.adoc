The `StateContext` interface provides a read-only view of the engine's current lifecycle state. It is primarily used by the `xref:../modules/EventEmitter.adoc[EventEmitter]` to determine which contextual channels to notify.

== Methods

[cols="1,3"]
|===
|Method |Description
|`isOn()` |Returns `true` if the system is powered on.
|`isStarted()` |Returns `true` if a game session has begun.
|`isPlaying()` |Returns `true` if gameplay is active and not paused.
|`isPaused()` |Returns `true` if the game is started but halted.
|`isGameOver()` |Returns `true` if the player has lost.
|===

== General Usage

This interface is implemented by `xref:../modules/GameState.adoc[GameState]` and is consumed by `notifyContextual` in the `EventEmitter`.
