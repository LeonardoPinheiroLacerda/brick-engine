== Architectural Purpose

The `GameSession` module implements the engine's core persistence logic orchestrator for transient serialized data. Rather than mixing user preferences (`GameState`) with dynamic game-over progress (like active piece locations or multipliers), `GameSession` secures an isolated boundary that hooks directly into the logical `xref:GameState.adoc[GameState]` phase shifts and intelligently pauses flow via modal confirmations before accidental memory leaks or overwrites occur.

== Session Management

=== `saveSession(): void`

Persists the current state of all registered `xref:../Serializable.adoc[Serializable]` instances. Early returns if the session is disabled (via `setSessionEnabled`) or if the session restoration flow is not yet complete (`isSessionResolved()` is `false`).

'''

=== `isSessionResolved(): boolean`

Implementation of the interface method. Returns whether the session resolution has been finalized.

'''

=== `setResetFunction(resetFn: () => void): void`

Stores the reset callback that will be invoked if the user chooses to cancel restoration.

'''

=== `setSessionEnabled(enabled: boolean): void`

Enables or disables the session recovery mechanism. When disabled, the engine will neither show the session recovery modal nor save the session.

**Parameters**

[cols="1,1,3"]
|===
|Name |Type |Description
|`enabled` |`boolean` |The primitive strict boolean enforcing if a startup evaluates past history (`true`) or resets blindly (`false`).
|===

=== `clearSession(): void`

Removes the saved instances from `localStorage`.

=== `syncState(state: State): void`

Subscribes to `xref:GameState.adoc[GameState]` interactions.

**Parameters**

[cols="1,1,3"]
|===
|Name |Type |Description
|`state` |`xref:../interfaces/modules/State.adoc[State]` |The global singleton source-of-truth object orchestrator explicitly exposing lifecycle events.
|===

**Behavior**

Reacts to:
- `PLAYING`: If a session for the current `gameId` exists, session is enabled, and the modal hasn't been closed, it invokes the modal via `_showSessionModal`. If accepted, calls deserialize on all registered targets. If canceled or if no previous session exists, proceeds normally. The modal state is tracked to prevent re-triggering during the same power cycle. If canceled, it also invokes the registered `resetFn` to ensure the game starts with a clean state.
- `ON` (when false): Resets the modal state tracking closure, allowing the session modal to appear again after the device is powered back on.
- `GAME_OVER`: Clears active session entirely.

== Internals

The system scopes data internally via the `${gameId}::${serialId}` key pattern within the `localStorage`. Under the hood, the engine identifies instances that implements the `xref:../interfaces/Serializable.adoc[Serializable]` capability and registers them here implicitly during the bootstrap.
