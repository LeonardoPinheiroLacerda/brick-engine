The `GameScore` module acts as the strict authority dedicated to tracking real-time player performance metrics, levels, and point multipliers. It prevents transient session logic from interacting with persistent storage, explicitly taking over saving the global `highScore` into an isolated `LocalStorage` namespace directly, ensuring different game engines do not overwrite each other's records.

It exposes reactivity through an event-based system using `xref:../enums/ScoreProperty.adoc[ScoreProperty]`.

== API Reference

=== Life Cycle

Methods for initializing the high score context.

==== `setupGameHighScore(id: string): void`

Initializes the module with the specific game ID and loads the corresponding high score.

**Parameters**

[cols="1,1,3", options="header"]
|===
|Name |Type |Description
|`id` |`string` |The unique game UUID identifier used as a persistent prefix for namespace isolation.
|===

**Example**

[source,typescript]
----
core.score.setupGameHighScore("tetris-v1");
----

'''

=== Score Management

Methods for adding and resetting points.

==== `score: number`

Read-only property returning the current session's points.

**Example**

[source,typescript]
----
console.log(`Current Score: ${core.score.score}`);
----

'''

==== `increaseScore(amount: number): void`

Adds points to the current session.

**Parameters**

[cols="1,1,3", options="header"]
|===
|Name |Type |Description
|`amount` |`number` |Base raw integer points to add before multiplier application.
|===

**Behavior**

Increases the score by `amount * multiplier`. Automatically compares the new score against the internal `highScore` property and updates it if a new record is reached (which in turn persists the new record to `LocalStorage`). Triggers a `SCORE` notification via the `xref:EventEmitter.adoc[EventEmitter]`.

**Example**

[source,typescript]
----
// Adds 500 * current multiplier to the score
core.score.increaseScore(500);
----

'''

==== `resetScore(): void`

Resets the session score to zero. Triggers a `SCORE` notification.

**Example**

[source,typescript]
----
core.score.resetScore();
----

'''

==== `highScore: number`

Read and Write property for the highest overall score achieved.

**Behavior**

When set, it automatically persists the value directly to `LocalStorage` under the namespaced key `${gameId}::highScore`. This ensures that different games do not overwrite each other's records. Triggers a `HIGH_SCORE` notification.

**Example**

[source,typescript]
----
console.log(`High Score to beat: ${core.score.highScore}`);
----

'''

=== Level Management

Methods for difficulty scaling.

==== `level: number`

Read-only property returning the current difficulty level.

**Example**

[source,typescript]
----
if (core.score.level > 10) {
  unlockSecretLevel();
}
----

'''

==== `increaseLevel(amount: number): void`

Increases the game difficulty level.

**Parameters**

[cols="1,1,3", options="header"]
|===
|Name |Type |Description
|`amount` |`number` |Positive integer count of step levels to advance globally.
|===

**Behavior**

Increases the internal level and triggers a `LEVEL` notification.

**Example**

[source,typescript]
----
core.score.increaseLevel(1);
----

'''

==== `resetLevel(): void`

Resets the level counter to 1. Triggers a `LEVEL` notification.

**Example**

[source,typescript]
----
core.score.resetLevel();
----

'''

==== `maxLevel: number`

Read and Write property for the maximum game level cap. Triggers a `MAX_LEVEL` notification on change.

**Example**

[source,typescript]
----
core.score.maxLevel = 25;
----

'''

=== Subscriptions

Enables other modules to listen for score-related updates.

==== `subscribe(callback: (score: number) => void): void`

Registers a listener for score changes.

**Parameters**

[cols="1,1,3", options="header"]
|===
|Name |Type |Description
|`callback` |`(score: number) => void` |The function to execute when the score changes.
|===

**Example**

[source,typescript]
----
core.score.subscribe((val) => hud.refresh(val));
----

'''

==== `unsubscribe(callback: (score: number) => void): void`

Removes an existing score listener.

**Parameters**

[cols="1,1,3", options="header"]
|===
|Name |Type |Description
|`callback` |`(score: number) => void` |The function to remove.
|===

**Example**

[source,typescript]
----
core.score.unsubscribe(onScoreUpdate);
----

'''

=== Utility & State

Methods for formatting and serialization.

==== `multiplier: number`

Read and Write property for the current points multiplier. Triggers a `MULTIPLIER` notification on change.

**Example**

[source,typescript]
----
core.score.multiplier = 2.5; // Enable power-up
----

'''

==== `getFormattedScore(digits: number): string`

Returns the score as a padded string suitable for HUD display.

**Parameters**

[cols="1,1,3", options="header"]
|===
|Name |Type |Description
|`digits` |`number` |Total character length to dictate string sequence width (e.g., 6 outputs "000150").
|===

**Returns**

`string` - The zero-padded score string.

**Example**

[source,typescript]
----
const label = core.score.getFormattedScore(6);
----

'''

==== `serialize(): string`

**Behavior**

Implements `xref:../interfaces/Serializable.adoc[Serializable]`. Packs the current `score`, `level`, `max_level`, `multiplier`, and `highScore` into a JSON string to be managed by the active game session.

**Example**

[source,typescript]
----
const saveData = core.score.serialize();
----

'''

==== `deserialize(data: string): void`

**Behavior**

Implements `xref:../interfaces/Serializable.adoc[Serializable]`. Restores all progress from a saved session string.

**Example**

[source,typescript]
----
core.score.deserialize(savedState);
----

'''

==== `getDebugData(): Record<string, string | number | boolean>`

Returns metadata for the real-time debugger.

**Example**

[source,typescript]
----
const debugInfo = core.score.getDebugData();
----
