The `EventEmitter` is the backbone of the engine's reactive architecture. It allows modules to communicate without direct dependencies and supports **Contextual Notifications** that automatically target specific game states (e.g., Playing vs. Paused).

== Contextual Notification System

The `EventEmitter` extends simple pub/sub by supporting event suffixes. When a module notifies a "contextual" event, the emitter propagates the signal to:
1.  The **Base Channel** (e.g., `score-changed`).
2.  An **Automatic Context Channel** (e.g., `score-changed:playing`).

This allows modules like `GameControl` to subscribe to keys ONLY when the game is in a specific state, preventing input leaking and simplifying conditional logic.

'''

== API Reference

=== `subscribe(eventName: string, callback: EventCallback<T>, suffix?: EventSuffix): void`

Registers a listener for a specific event.

**Parameters**

[cols="1,1,3"]
|===
|Name |Type |Description
|`eventName` |`string` |The base name of the event.
|`callback` |`Function` |The function to execute.
|`suffix` |`xref:../enums/EventSuffix.adoc[EventSuffix]` |(Optional) The context suffix (e.g., `EventSuffix.PAUSED`).
|===

'''

=== `unsubscribe(eventName: string, callback: EventCallback<T>, suffix?: EventSuffix): void`

Removes a registration. The suffix must match the one used during subscription.

'''

=== `notify(eventName: string, payload: T): void`

Dispatches an event to the base channel.

'''

=== `notifyContextual(eventName: string, payload: T, context: StateContext): void`

Dispatches an event to the base channel AND an appropriate state-specific channel.

**Parameters**

[cols="1,1,3"]
|===
|Name |Type |Description
|`eventName` |`string` |The base name.
|`payload` |`any` |The data to pass.
|`context` |`xref:../interfaces/StateContext.adoc[StateContext]` |The current engine context.
|===

**Mapping Logic**

*   `context.isPaused()` -> `eventName:paused`
*   `context.isPlaying()` -> `eventName:playing`
*   `context.isGameOver()` -> `eventName:gameover`
*   `context.isOn() && !context.isStarted()` -> `eventName:title`

'''

== Internal Name Formatting

The `EventEmitter` encapsulates its naming convention within a private `_formatName` method. The current standard uses a colon separator:

`[baseName]:[suffix]` (e.g., `UP:pressed:playing`)

Modules should never manually format these strings; they should use the `suffix` parameter in the `subscribe` API.
