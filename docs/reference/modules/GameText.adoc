== Architectural Purpose

The `GameText` module isolates all typography and sizing logic away from the rendering pipelines. By pre-calculating relative font sizing constraints and wrapping absolute rendering parameters over internal relative metrics structures (`xref:../types/Coordinate.adoc[Coordinate]`), it guarantees that the HUD or grid components do not have to directly calculate native element boundaries against the volatile physical display size.

== API Reference

=== `setTextSize(size: FontSize): void`

Sets the current font size from predefined responsive values.

**Parameters**

[cols="1,1,3"]
|===
|Name |Type |Description
|`size` |`xref:../enums/FontSize.adoc[FontSize]` |The preset enum scalar value representing the responsive dimension required.
|===

**Behavior**

Applies the pre-calculated pixel size to the P5.js context based on the current container dimensions.

**Example**

[source,typescript]
----
this.modules.text.setTextSize(FontSize.LARGE);
----


'''

=== `writeOnDisplay(text: string, coordinate: Coordinate): void`

Renders text on the main game display area.

**Parameters**

[cols="1,1,3"]
|===
|Name |Type |Description
|`text` |`string` |The direct formatted string intended for output.
|`coordinate` |`xref:../types/Coordinate.adoc[Coordinate]` |Floating point proportional coordinate anchor points (between `0.0` and `1.0`) relative exclusively to the parent grid bounds.
|===

**Example**

[source,typescript]
----
text.writeOnDisplay('PAUSED', { x: 0.5, y: 0.5 });
----


'''
