== Architectural Purpose

The `GameTime` module is designed to decouple deterministic game logic "ticks" from the unpredictable visual frame rate of the browser. By maintaining an internal time accumulator, it ensures that gameplay events (like physics, timers, and difficulty scaling) execute at a consistent speed regardless of the host device's hardware performance or FPS fluctuations.

== API Reference

=== Core Loop Control

These methods are used by the engine's main loop to synchronize logical state updates with the physical rendering time.

==== `update(deltaTime: number): void`

Calculates the current physical performance metrics (FPS/TPS). This method should be called exactly once per visual frame.

**Parameters**

[cols="1,1,3"]
|===
|Name |Type |Description
|`deltaTime` |`number` |The physical time elapsed since the last rendering frame in milliseconds.
|===

'''

==== `accumulate(deltaTime: number): void`

Adds elapsed time to the logical simulation accumulator. This allows the engine to track how much logical time needs to be processed.

**Parameters**

[cols="1,1,3"]
|===
|Name |Type |Description
|`deltaTime` |`number` |Time in milliseconds to add to the logic simulation.
|===

'''

==== `shouldTick(): boolean`

Evaluates if the engine should execute a logical simulation tick based on the accumulated time.

**Returns**

`boolean` - `true` if at least one tick's worth of time was consumed from the accumulator.

**Behavior**

When this method returns `true`, it has already internally consumed all pending ticks from the `_tickAccumulator` to prevent "speed-up" bursts after lag, while allowing the caller to perform a single logical update.

'''

==== `reset(): void`

Resets all internal timers, statistical accumulators, and logical counters to their baseline values.

**Behavior**

Prevents a "burst" of logic updates after resuming from a long pause or game restart by clearing the time accumulator.

'''

=== Declarative Timing

These methods provide a high-level API for developers to schedule events based on logical game time rather than raw frame counts.

==== `atInterval(ms: number): boolean`

Checks if the logical game clock has just crossed a multiple of the specified millisecond interval.

**Parameters**

[cols="1,1,3"]
|===
|Name |Type |Description
|`ms` |`number` |The millisecond interval to check against.
|===

**Returns**

`boolean` - `true` if the logical clock just crossed a boundary of the interval.

**Example**

[source,typescript]
----
if (time.atInterval(500)) {
    this.blink();
}
----

'''

==== `every(ms: number, action: () => void): void`

Executes a callback function every N milliseconds of logical game time.

**Parameters**

[cols="1,1,3"]
|===
|Name |Type |Description
|`ms` |`number` |The millisecond interval between executions.
|`action` |`() => void` |The callback function to execute.
|===

**Example**

[source,typescript]
----
time.every(1000, () => this.spawnEnemy());
----

'''

'''

'''
