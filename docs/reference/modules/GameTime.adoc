== Architectural Purpose

The `GameTime` module manages the engine's logical clock independent of the visual context. By decoupling the deterministic game logic "ticks" from the unpredictable visual frame rate of the browser, it ensures consistent gameplay speed and difficulty-progression physics regardless of the device's hardware performance capabilities.

== API Reference

=== `tickInterval: number`

Getter/Setter for the game logic speed in milliseconds.

**Returns**

`number` - The current time between logic ticks.

**Behavior**

Lowering this value makes the game run faster (e.g., used for difficulty progression).


'''

=== `totalTicks: number`

Read-only property returning the total number of logic ticks that have occurred since the game started.


'''

=== `elapsedTime: number`

Read-only property returning the total time in milliseconds elapsed since the game started.


'''

=== `shouldTick(): boolean`

Determines if a logic update should be executed.

**Returns**

`boolean` - `true` if the internal ticker buffer (accumulator) has exceeded the `tickInterval`.

**Behavior**

Called by the main loop. If it returns true, it automatically subtracts the `tickInterval` from the `tickAccumulator` to maintain a stable clock.


'''

=== `update(deltaTime: number): void`

Advances the internal time buffer (tick accumulator).

**Parameters**

[cols="1,1,3"]
|===
|Name |Type |Description
|`deltaTime` |`number` |Time physically elapsed since the last rendering frame (in milliseconds).
|===


'''

=== `reset(): void`

Clears the tick accumulator.

**Behavior**

Prevents a "burst" of logic updates after resuming from a long pause or game restart.


'''

=== `isTickEvery(interval: number): boolean`

Returns `true` if the `totalTicks` is a multiple of the specified `interval`. Useful for scheduling recurring game events (e.g., spawning an item every 20 ticks).

**Parameters**

[cols="1,1,3"]
|===
|Name |Type |Description
|`interval` |`number` |The tick frequency interval constraint to check against.
|===

**Returns**

`boolean` - `true` if the current tick is a multiple of the interval (and greater than 0).


'''

=== `incrementTickInterval(amount: number): void`

Slows down the game by increasing the time between ticks.

**Parameters**

[cols="1,1,3"]
|===
|Name |Type |Description
|`amount` |`number` |Milliseconds to add to the interval.
|===


'''

=== `decrementTickInterval(amount: number): void`

Speeds up the game by decreasing the time between ticks.

**Parameters**

[cols="1,1,3"]
|===
|Name |Type |Description
|`amount` |`number` |Milliseconds to subtract from the interval.
|===


'''

=== `setTickInterval(interval: number): void`

Sets the tick interval exactly to the specified value.

**Parameters**

[cols="1,1,3"]
|===
|Name |Type |Description
|`interval` |`number` |The precise new integer interval interval between logic updates (in milliseconds).
|===


'''

=== `setMinTickInterval(interval: number): void`

Sets the minimum limit for the tick interval, preventing the game from speeding up beyond this point.

**Parameters**

[cols="1,1,3"]
|===
|Name |Type |Description
|`interval` |`number` |The strict minimum boundary interval allowed (in milliseconds).
|===


'''

=== `getDebugData(): Record<string, string | number | boolean>`

Returns metadata for the real-time debugger.

**Returns**

`Record<string, string | number | boolean>` - Object containing `fps` (Frames Per Second), `tps` (Ticks Per Second), `tick_interval`, and `tick_accumulator`.


'''

=== `captureInitialState(): void`

Captures the current `tickInterval` as the starting speed for the game session.

**Behavior**

This method is called by the `Game` engine after the game-specific `setupGame()` has executed. It ensures that when `reset()` is called, the game returns to its intended initial speed rather than the engine's global default.
