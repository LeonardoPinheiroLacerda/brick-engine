== Architectural Purpose

The `InitialStateSnapshot` acts as an agnostic structural utility handling transparent auto-save boundaries. Rather than forcing developers to write boilerplate serialization functions for their native sub-classes, the Engine relies on this module to dynamically iterate, deeply clone, and selectively safely cache customized attributes into local memory buffers to restore configurations effortlessly upon reset triggers.

== Methods

=== `captureBaseProperties`

Captures the base properties of the instance before any custom client properties are added. In the engine's architecture, this should be called at the very end of the base `Game` constructor, establishing the baseline for what should be ignored during the snapshot phase automatically.

[cols="1,1,3", options="header"]
|===
|Parameter |Type |Description
|`instance` |`object` |The object instance to extract keys from.
|===

'''

=== `captureInitialState`

Stores the initial state of custom client properties. It selectively clones properties that were not present in the base properties and are not functions. The deep cloning strategy involves:
1. Checking if the object has a custom `clone()` method.
2. Attempting to use `structuredClone` for plain objects.
3. Safely falling back to by-reference assignment for complex objects like P5 Vectors if other methods fail.

[cols="1,1,3", options="header"]
|===
|Parameter |Type |Description
|`instance` |`object` |The object instance containing custom properties.
|===

'''

=== `restoreInitialState`

Restores the captured custom properties back to the instance, effectively resetting the client's state.

[cols="1,1,3", options="header"]
|===
|Parameter |Type |Description
|`instance` |`object` |The object instance to restore properties onto.
|===
